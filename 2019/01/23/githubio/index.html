github pages + hexo搭建简易个人博客

准备部分

概述

博客的建立分为两部分，其一是github_pages。其二是hexo。

github-pages是github的一个服务，可以建立自己的网站

hexo用来创建一个网站。它基于node.js，需要先下载node.js的环境。之后使用npm下载hexo的安装包。将其安装后，修改配置，安装插件即可使用它在github上显示博客页面。

整体流程

  1. 环境配置,包括node.js,npm,git下载及配置
  2. github_pages的创建
  3. 使用npm下载hexo并安装
  4. 使用hexo初始化一个目录
  5. _config.yml的配置以及博客部署插件的安装
  6. 编辑博客并发布

一、环境配置

  1. 安装node.js及npm、git

  2. 配置github上ssh密钥对,使得本地可以ssh连接github

    使用下面的命令即可查看是否已经配置成功

    ssh -T git@github.com

二、github_pages的创建

在github中创建一个新的资源库，将其命名为your_github_name.github.io即可。创建后，即可在settings界面下看到github_page处显示相关信息

三、hexo相关操作

下载hexo

有两种方法

  1. 如果安装了npm,则可以使用npm下载hexo并安装hexo
下载：

    npm install hexo(-cli) -g

安装：

    npm install hexo --save

hexo的使用

将hexo源文件初始化到相应目录

  1. 初始化Hexo。执行下目标文件夹会生成源文件

    hexo init

  1. hexo服务器的启动:

    hexo s（erver）

    出现提示：

    INFO Hexo is running at http://0.0.0.0:4000/. Press Ctrl+C to stop.

    在浏览器中打开http://localhost:4000/即可查看本地搭建的hexo博客，到此可以判断hexo安装成功

  2. 修改配置文件_config.yml的部分内容如下

    deploy:
  type: git
  repo: git@github.com:yourname/yourname.github.io.git
  branch: master

  3. 安装将本地内容部署到github所需要的插件

    npm install hexo-deployer-git --save

使用hexo编辑、发布博客

  1. 写博客

    hexo new post "your-article-name"

    这时在/hexo/source/_posts 将会看到 your-article-name.md 文件用MarDown编辑器打开就可以编辑文章了。

  2. 博客上传（正式把博客上传到github上，并可以网络访问）

    依次执行以下命令：清理 – 生成 – 部署

    $ hexo clean        //清除hexo缓存
$ hexo g(enerate)   // 生成
$ hexo d(eploy)     // 部署

    也可以执行下面的命令，相当于上面两条命令的效果：

    hexo d -g #在部署前先生成
